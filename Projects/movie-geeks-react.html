Movie-Geeks Client-Side Web App

Project Overview

The Movie-Geeks client-side is a React web application styled for classic film enthusiasts. The front end is built with Parcel and styled with SCSS and Bootstrap. It connects to the Movie-Geeks API to fetch movie data and manage user profiles.
Users are able to:

  • View movie data, including details about genres, directors, and actors
  • Register and manage user accounts
  • Add or remove favorite movies

Technologies Used

  • React
  • Parcel
  • SCSS via @parcel/transformer-sass.
  • Bootstrap and Bootstrap Icons
  • React Router DOM
  • JWT Authentication (handled by the Movie-Geeks API backend)
  • Netlify for app hosting

Objective

The sister project to Movie-Geeps REST API. The challenge was to build a React Web App, responsively designed with Bootstrap, allowing users to sign in or sign up, browse their favourite movies, save or remove movies in a favourites list, update user information (Username, email, password).

The Experience

I found this project to be of interest for a number of reasons. I learned through my tutor it is not just technical reasons that dictate why you use a certain framework. Business value, Customer Experience and Good Engineering Practices are what really count. Having said that, React is a great framework to learn with its component structure and use as its popularity is undeniable. The use of outdated technologies in the course gave me some self learning opportunities when trying to find solutions. For styling, I chose classic European films in my database which led me to an Art Deco inspired look, in the style of retro cinemas.

The process included:

  • Creating React components using JSX
  • Render Data from the Movie-Geeks API
  • Execute frontend form validation with React and Apply authentication logic to the form
  • Style the app using the React Bootstrap library
  • Installing and implementing Redux managed states to all views
  • Deploy the app using Netlify
