Movie-Geeks Server Side Component

Project Overview

The Movie Geek REST API is the backend for the Movie Geeks web application (And the client side Angular web App "myFlix"). It provides endpoints for users to:

  • View movie data, including details about genres, directors, and actors
  • Register and manage user accounts
  • Add or remove favorite movies

Technologies Used

  • Node.js
  • MongoDB - for managing user and movie data
  • Express
  • Vercel - for backend deployment
  • JWT Authentication via Passport
  • Postman for testing

Objective

The objective of the project was to build a REST API for a “Movies” web application where users can receive information about their favourite movies, the director and genre and to create a profile where they can save a listing of their favourite movies.

The Experience

Creating this REST API gave me a better understanding of databases and the Vercel serverless cloud platform. It allowed me to become more comfortable using the terminal and introduced me to frameworks (Specifically the Express framework) and URL endpoints alongside CRUD (Create, Read, Update and Delete) operations, testing the API with Postman, implementing CORS and Hashing passwords . All in all a great backend learning experience.

The process included:

  • Using and practising Node,js syntax
  • Creating package.json file and importing necessary dependencies
  • Using Route HTTP requests and defining endpoints for the Rest API
  • Creating a relational(SQL) database and a non-relational(NoSQL) database using MongoDB
  • Modelling business logic using Mongoose
  • Implement authentication and authorization using basic HTTP authentication and JWT (token-based) authentication
  • Incorporate data validation logic
  • Implement data security and storage controls
  • Hosting the project on the web using Vercel
