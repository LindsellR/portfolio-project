The Meet App

Project Overview

A serverless progressive web application (PWA) built using a test driven development (TDD) approach and utilising a Google Calendars API to fetch events. The app uses OAuth2 authentication and AWS serverless functions for the authorisation server.

Technologies Used

  • React
  • Google Calendar API
  • AWS Lambda
  • OAuth2 Authentication
  • Rechart Charting Library
  • Jest-Cucumber
  • Recharts charting library

Objective

To build a serverless progressive web app (PWA) that allows users to search for events in a nearby city, show and hide event details, specify the number of events displayed, operate while offline and visually display event details in a chart.

The Experience

This project was a great introduction to a Test-Driven Development (TDD) and Behavior-Driven Development (BDD) approach in building an app. Debugging is the name of the game with TDD and this gave me an insight into how this process streamlines and speeds up the development of an app even though intuitively you think it would slow things down. By unit and integration testing with Jest and BDD testing with Jest-Cucumber it gave me an insight into how important user scenarios are in the development process.

The process included:

  • Creating user stories based on user requirements and app functionality.
  • Translating user stories into multiple test scenarios using Jest-Cucumber.
  • Writing unit tests for each function.
  • Implementing and testing integration use cases.
  • Writing acceptance tests for non-developers and conducting automated end-to-end tests.
  • Implementing Progressive Web App (PWA) features to support offline use.
  • Visualizing event data using the Recharts charting library
